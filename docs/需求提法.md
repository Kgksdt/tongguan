# 需求提法

你后面给我提需求时，最好按这个顺序说。越具体，我越能一次做对。

## 一句话目标

说明这次要做什么。

示例：

> 我要做一个算法知识点通关页面，能看到 CSP-S 到 CF2000 的所有知识点，以及每个知识点当前状态。

## 使用对象

说明谁会用这个功能。

可选：

- 我自己学习用
- 我备课用
- 给学生看
- 给家长看
- 机构内部管理用

## 页面或功能

说明这次要做哪一块。

示例：

- 首页仪表盘
- 知识地图
- 题单管理
- 错题复盘
- 教师评价模板
- 学生反馈生成

## 数据字段

说明每条数据要保存什么。

示例：

知识点需要：

- 名称
- 阶段
- 难度
- 状态
- 通关标准
- 推荐题目
- 复习日期

## 操作流程

说明用户怎么点、怎么填、怎么得到结果。

示例：

> 我进入知识地图，点击“背包 DP”，右侧出现通关标准和题单。我可以把状态改成“练习中”，也可以新增一道题，填入链接、难度、结果和错因。

## 界面感觉

说明你希望它长什么样。

示例：

- 像教师工作台，清爽、严肃、信息密度高
- 不要花哨
- 移动端能看，但主要给电脑用
- 重点突出进度和下一步任务

## 验收标准

说明做到什么程度算完成。

示例：

- 能本地打开
- 页面不报错
- 能新增、编辑、删除知识点
- 刷新后数据还在
- 至少内置 CSP-S 核心知识点

## 推荐格式

直接复制这个模板：

```text
这次我要做：

使用对象：

页面/功能：

需要保存的数据：

用户操作流程：

界面风格：

验收标准：
```

## 一个完整例子

```text
这次我要做：知识地图页面。

使用对象：我自己学习和备课用。

页面/功能：左边按阶段显示 CSP-S、提高、CF1600、CF1800、CF2000；中间显示知识点卡片；右边显示选中知识点的通关标准。

需要保存的数据：知识点名称、阶段、状态、通关标准、推荐题、错题数量、下次复习日期。

用户操作流程：点击知识点后，右侧出现详情；可以修改状态；可以勾选通关标准；可以添加题目记录。

界面风格：清爽、像教师工作台，不要花哨，电脑端优先。

验收标准：页面能运行，内置至少 30 个知识点，状态可以切换，刷新后数据不丢。
```
